# Charts

Charts are generated by matplotlib scripts, saved to `assets/`, and embedded via `<image>` like any other PNG. Don't try to draw charts as inline SVG primitives.

## Why

- Matplotlib handles axes, ticks, legends, log scales, error bars — re-implementing in SVG is bottomless.
- Charts get re-generated when data changes; scripts diff cleanly.
- The slide SVG stays focused on layout, not data binding.

## Pattern

### 1. Chart script

`<slides-root>/scripts/slide<N>-<name>.py`:

```python
from pathlib import Path
import matplotlib
matplotlib.use("Agg")
import matplotlib.pyplot as plt

# Palette must match the slide's theme. Hard rule: never invent colors.
# Map matplotlib constants → theme CSS classes:
#   FIGURE_BG → .bg          fill
#   AXES_BG   → .surface-alt fill
#   TEXT      → .text        fill
#   MUTED     → .text-muted  fill
#   SPINE     → .border      stroke
#   BAR       → .accent      fill
FIGURE_BG = "#ffffff"
AXES_BG   = "#fafbfc"
TEXT      = "#1a1a1a"
MUTED     = "#666666"
SPINE     = "#e5e7eb"
BAR       = "#0078d4"

DATA = [("label_a", 0.12), ("label_b", 0.08), ...]

def build():
    out_dir = Path(__file__).resolve().parent.parent / "assets"
    out_dir.mkdir(parents=True, exist_ok=True)

    fig, ax = plt.subplots(figsize=(11, 7.2), dpi=100, facecolor=FIGURE_BG)
    ax.set_facecolor(AXES_BG)
    ax.barh(range(len(DATA)), [v for _, v in DATA], color=BAR)
    # ... ticks, labels, spines styled with TEXT / MUTED / SPINE ...

    out = out_dir / "slide<N>-<name>.png"
    fig.savefig(out, dpi=100, bbox_inches="tight", facecolor=FIGURE_BG)
    plt.close(fig)
    print(out)

if __name__ == "__main__":
    build()
```

### 2. Embed in slide

```xml
<image
  href="./assets/slide5-text-robust-results.png"
  xlink:href="./assets/slide5-text-robust-results.png"
  x="60" y="220" width="1100" height="700"
  preserveAspectRatio="xMidYMid meet" />
```

Use `meet` for charts so axis labels don't get clipped.

### 3. Re-render after data change

```bash
python3 scripts/slide5-<name>.py
rsvg-convert -w 1920 -h 1080 slide-05-<name>.svg -o renders/slide-05.png
```

## Color palette must match the theme

If the slide is light theme, the chart must be light theme. The palette constants in the chart script duplicate the theme CSS — that's accepted duplication for renderer-isolation reasons.

When you switch a slide's theme via `apply-theme.sh`, you must also regenerate its charts with the matching palette.

## Sizing

- **Wide single chart on a slide:** `figsize=(11, 7.2)`, embedded at `width="1100"` `height="700"`.
- **Half-slide chart (chart + bullets):** `figsize=(8, 6)`, embedded at `width="900"` `height="650"`.
- **Stacked half-height (two charts on one slide):** `figsize=(11, 4.5)`, embedded at `width="1100"` `height="380"`.

Always use `dpi=100`; the SVG scales the embedded raster, no need for higher density.

## Anti-patterns

- Drawing chart bars as `<rect>` elements in the SVG directly — fine for a sparkline, brutal for anything with axes.
- Re-running the script every render — only re-run when data or palette changes; the PNG is cached.
- Using `matplotlib.pyplot` colormaps (`viridis`, `tab10`) instead of theme colors — chart will clash with the slide.
- Embedding the matplotlib SVG output instead of PNG — sometimes the matplotlib SVG has font issues in rsvg/PPT; PNG is reliable.
