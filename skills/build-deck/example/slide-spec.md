## Slide 2 — Bullet lists
- **Layout:** title + 4 bullets
- **Headline:** Bullet lists
- **Subtitle:** One `<text>` block. `dx` after the marker. `dy` between rows.
- **Content blocks:**
  - **Bullets:**
    - Markers are `•` glyphs in `.accent`; body text in `.body`.
    - `dx="16"` spaces the body away from the bullet glyph.
    - `dy="1.5em"` on rows 2+ advances one line per bullet.
    - Inline emphasis with a `body-bold text` tspan — use sparingly.
- **Reference:** `references/pattern-bullets.md`
- **Talking points:**
  - Bullets use the dy pattern because one `<text>` per row causes drift when you add or remove items.
  - The `dx="16"` value offsets the body text from the bullet glyph; adjust it if you change the glyph width.
  - Inline bold with `body-bold` is intentional here — call it out as a structural rule, not decoration.
- **Speaker notes:**
  - dy pattern, no bullet drift on edit
  - dx=16 is the body offset
  - bold = structural, not decorative
- **Status:** ✅

## Slide 5 — Matplotlib charts
- **Layout:** title + centered chart, palette-matched caption
- **Headline:** Matplotlib charts
- **Subtitle:** Python generates the chart. Palette pulled from theme CSS. Embedded via `<image>` with both `href` and `xlink:href`.
- **Visuals:** `assets/slide5-perturbation-chart.png` — horizontal bar chart of perturbation impact
- **Reference:** `references/pattern-charts.md`
- **Talking points:**
  - The chart is generated by a Python script that reads the palette from the theme CSS, so colors automatically match a theme swap.
  - Embed via `<image href>` with both `href` and `xlink:href` for renderer compatibility — rsvg-convert needs the former, some older PowerPoint builds need the latter.
  - The build script inlines the PNG as a base64 data URI, so the pptx has no external file dependencies.
- **Speaker notes:**
  - palette comes from theme CSS
  - use both href and xlink:href
  - PNG is inlined, no external deps
- **Status:** ✅

## Slide 6 — SVG flow diagrams
- **Layout:** title + flow diagram (boxes + arrows in native SVG)
- **Headline:** SVG flow diagrams
- **Subtitle:** Boxes are `<rect>`, arrows are `<line>` with `marker-end`. No chart pipeline, no raster.
- **Content blocks:**
  - 4-box flow: `slide-NN.svg` → branches into `rsvg-convert → renders/*.png` (authoring) and `build-pptx.py → deck.pptx` (delivery)
- **Talking points:**
  - Native SVG flow diagrams use `<rect>` and `<line>` — no chart library, no raster, fully vector.
  - Arrows use `marker-end` with a named `<marker>` element; define the marker once and reference it from every line.
  - This is the right choice when the diagram is small, the boxes are fixed layout, and you want the output to scale cleanly in PowerPoint.
- **Speaker notes:**
  - native SVG, no chart pipeline
  - define marker once, reuse on lines
  - best for fixed-layout diagrams
- **Status:** ✅
