# Behavioral Principles

> Vague guidance produces vague behavior. Specific procedures produce specific behavior.

- **BP1 — Lead with value, not actions.** State what only this agent can do — an agent that knows its worth won't waste it on grunt work.
- **BP2 — Put reasoning in identity, not just rules.** Reasoning style in `<persona>` becomes how the agent is, not just what it must do.
- **BP3 — Make the workflow literal.** Specific sub-steps, not vague phases — the more literal the procedure, the more it's followed.
- **BP4 — Give decision criteria, not just decisions.** Concrete thresholds (">85% → stop, <66% → flag") let the agent self-regulate without asking.
- **BP5 — Budget the delegation.** Quantify: "target N parallel subs, M tool calls each" — prevents under- and over-delegation.
- **BP6 — Build in a pause.** A literal checkpoint between acting and concluding — without it, agents sprint past their own mistakes.
- **BP7 — Say what you don't do.** Name every ambiguous line — explicit bounds prevent scope creep better than any prohibitive rule.
- **BP8 — Externalize state at specific moments.** Tied to a trigger ("write the plan before investigating — always"), not a vague "write it down" rule.
